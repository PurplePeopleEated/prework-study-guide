Prework Study Guide

HTML

  • is used to create the structure of a webpage
  • is made up of elements that combine to provide content and basic framework for all webpages
  • element: type of content on a webpage
    • common element types:
      • text elements
      • image elements
      • header elements
      • list elements
  • element types used to divide and describe a page
    • meta-related elements
    • content division elements
    • semantic elements
  • two elements important to understanding how the structure works: head & body elements
    • head element: contains info about the webpage
    • body element: represents the visible content shown to the user
  • metadata: data about a webpage that the browser uses to serve a webpage to users
  • common meadata tags for head element:
    • title: absolutely required(especially for SEO) and defines the title for the webpage
    • style: defines the CSS styles asociated with a website
    • base: defines the base URL for a webpage
    • link: connects an external resource to the HTML document
    • meta: defines metadata like the character set(charset), HTTP header for content(http-equiv)description, keywords, author, and viewport
    • script: embeds client-side scripts in the HTML document

CSS

  • aka Cascading Style Sheets
  • is used to define how content is presented to users
  • three ways to style using CSS:
    1. inline CSS: applies to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only
    2. internal CSS style sheet: contains CSS rules for the webpage in the head section of the HTML document that only applies to that page
    3. external CSS style sheet: all CSS rules are kept in a separate file(preferred by most developers)
  • separation of concerns: separating code into different files based on use
  • a margin indicates how much space we want around the outside of an element
  • a padding indicates how much space we want around the content inside an element

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • GitHub Flow:
    1. create a feature branch
    2. make commits of all the changes we make to the code while working with a descriptive message attached
    3. push changes to the feature branch in the GitHub repo with a PR or pull request
    4. once PR is opened, others on your team can evaluate your code, and you can resolve conflicts between branches
    5. once PR is approved, it can be merged with the main branch and becomes the new main branch

JavaScript

  • a dynamic language that allows users to interact with websites
  • variable: a named container that allows us to store data in our code
    • example: var person = "alive";
    • the variable is named (person), the assignment operator (=) says we want to store the data ("alive") in (person)
  • string: always contained within quotation marks
  • console: a test environment that developers use to check out their code
  • method: a set of instructions that can be executed by a computer
    • example: .log() will output whatever is added inside the parentheses to the console
  • MAKE SURE TO LINK THE JAVASCRIPT FILE TO THE HTML DOCUMENT OR IT WON'T WORK
    • add this code under the footer element and above the closing element for the body:
      • (script src="add filepath here")(/script)
  • Chrome DevTools: a very important tool that has several capabilities
  • control flow: the order in which a comoputer executes code in a script
  • in JavaScript, code starts from the first statement at the top and ends at the last statement at the bottom
  • conditional statments and loops can be used to control the order in which code is executed or if it even is
  • if statement
    • code will only execute if the statement meets a condition/is true
    • you can have more than one condition by using else if
    • syntax for an if statement:
      • if (first condition) { block of code to be executed if first condition is truthy goes in here
      • } else if (second condition) { block of code to be executed if second condition is truthy goes in here }
    • === is a strict equality operator aka it checks to see if two values are equal and returns a Boolean result(true/false)
  • array: a single variable that is used to hold a group of data
  • arrays are mostly used for related data sets and can store different types of data
  • syntax for an array:
    • var variableName = [var1, var2, var3];
  • all arrays are indexed in the order named aka var1 = 0, var2 = 1, var3 = 2
  • length: command to pull the built-in property of an array
  • for loop: uses the predictable pattern of indices to perform a task on all the items in an array by allowing repeated exectution of a code block
    • contains three important statements:
      1. the starting point for our loop where we declare a variable and give it a value
      2. the condition: as long as the number in the associated variable is less than the length of the array, the loop will keep running
      3. the final statement is what allows the arrray to iterate over each items
        • the number of an index increases by 1 each time aka x+1
        • x++ is a shortcut way of x+1
        • every time the code loops, x will increase by 1
    • all code that we want to be executed for each loop is put inside curly brackets{}
    • for loop syntax:
      • for(var x = 0; x (less than carrot) variableName.length; x++) { code block goes in here }
  • function: a set of instructions that tells the computer how to perform a certain task
    • functions don't auto execute when the JavaScript file is run
    • they must be called by name in order to be executed
    • parentheses must be included whenever writing a function
    • syntax for a function named helloWorld:
      • function helloWorld() { code block goes in here }
    • Math: built-in object in JavaScript that can be used in various ways
    • Math.random() and Math.floor() can be used together to generate a random number
    • syntax for random number generation:
      • var variableName = Name[Math.floor(Math.random() * Name.length)]
